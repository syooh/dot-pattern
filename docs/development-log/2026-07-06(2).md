### 1. Undo / Redo 개선

- canUndo / canRedo 상태 추가

---

### 2. Tool 시스템 추가

Pattern Editor에서 사용할 Tool 시스템을 설계하였다.

현재 지원하는 Tool

- Brush
- Eraser
- Fill(준비 중)

ToolType을 별도의 타입으로 분리하여 관리하도록 변경하였다.

---

### 3. Brush / Eraser 연결

selectedTool 상태에 따라 PatternEngine의 함수를 호출하도록 수정하였다.

현재 동작

Brush → paintPixel()

Eraser → erasePixel()

Fill → 구현 예정

Canvas는 Tool을 알지 않고 usePattern이 모든 작업을 담당하도록 구조를 변경하였다.

---

### 4. 프로젝트 구조 리팩토링

usePattern 구조를 프로젝트 표준 구조로 정리하였다.

적용한 규칙

- import 순서 통일
- State → Computed → Function → Return 구조
- Return 순서 통일
- 주석 스타일 통일
- Boolean 변수 네이밍 규칙 적용

---

## 느낀 점

Tool 시스템을 먼저 설계한 덕분에 앞으로 Fill, Rectangle, Circle 등의 기능을 동일한 방식으로 쉽게 추가할 수 있는 구조가 되었다.

Undo / Redo와 Tool 시스템이 연결되면서 프로젝트의 기본 아키텍처가 완성 단계에 들어섰다.
