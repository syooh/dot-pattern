## 진행 내용

### 1. Palette 색상 삭제 기능 구현

사용자가 직접 추가한 색상을 Palette에서 삭제할 수 있는 기능을 구현하였다.

기본 색상(White, Black)은 삭제되지 않도록 제한하였다.

삭제된 색상을 사용 중인 픽셀은 기본색(White)으로 자동 변경되도록 PatternEngine에서 처리하였다.

---

### 2. PatternEngine 개선

PatternEngine에 `removeColor()` 함수를 추가하였다.

색상이 삭제될 경우

- Palette에서 색상 제거
- 모든 픽셀 검사
- 삭제된 색상을 사용하는 픽셀을 기본색으로 변경

하도록 구현하였다.

비즈니스 로직을 Engine으로 분리하여 React와 역할을 명확히 구분하였다.

---

### 3. History 구조 추가

Undo 기능을 위한 History 배열을 추가하였다.

모든 편집 작업 전에 현재 Pattern을 저장하는 구조를 설계하였다.

추가된 상태

- history
- future

---

### 4. saveHistory() 함수 구현

모든 편집 기능이 동일한 방식으로 Undo를 지원할 수 있도록

공통 함수인 `saveHistory()`를 구현하였다.

새로운 편집이 발생하면

- 현재 Pattern 저장
- Future(Redo) 기록 초기화

가 자동으로 수행된다.

---

### 5. 새 도안 생성 시 History 초기화

새 프로젝트 생성 시

- History
- Future

를 모두 초기화하도록 수정하였다.

이전 프로젝트의 Undo 기록이 남지 않도록 개선하였다.

---

### 6. usePattern 구조 개선

usePattern의 편집 흐름을 하나의 구조로 통일하였다.

모든 편집 기능이

saveHistory()

↓

PatternEngine

↓

setPattern()

순서로 동작하도록 리팩토링하였다.

현재 적용된 기능

- paintPixel()
- addColor()
- removeColor()

---

## 리팩토링

프로젝트 전체에서 사용할 편집 흐름을 표준화하였다.

기존

기능마다 다른 방식으로 Pattern을 수정

↓

변경

saveHistory()

↓

PatternEngine

↓

setPattern()

앞으로 추가될 모든 기능도 동일한 구조를 사용할 예정이다.

---

## 느낀 점

Undo 기능을 먼저 설계하면서 프로젝트의 전체 구조가 훨씬 명확해졌다.

편집 기능이 모두 동일한 흐름을 사용하도록 통일하여 유지보수성과 확장성이 크게 향상되었다.

앞으로 Tool 시스템을 추가하더라도 기존 구조를 변경하지 않고 기능만 추가할 수 있는 기반이 마련되었다.
